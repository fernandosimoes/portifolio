# Data Model: AI Chat Mode with Generative UI

**Feature**: 001-ai-chat-mode
**Date**: 2026-03-28

---

## Entities

### ModeState

Represents which interface the portfolio is currently displaying.

```
type ModeState = "terminal" | "chat"
```

- Lives in `page.tsx` as `useState<ModeState>("terminal")`.
- Passed down to `TerminalHeader` for rendering the active toggle.
- Controls which main view component (`TerminalView` | `ChatInterface`) is rendered
  inside the `AnimatePresence` wrapper.

---

### ChatMessage

A single message in the conversation. Follows the Vercel AI SDK `Message` type exactly
so it can be passed directly to `useChat` and serialised for the API route.

```
interface ChatMessage {
  id: string               // Unique identifier (generated by AI SDK)
  role: "user" | "assistant"
  content: string          // Plain text content (may be empty when parts present)
  parts?: MessagePart[]    // Structured content parts (text, tool-invocation)
  createdAt?: Date
}
```

The `parts` array is the primary rendering target:
- `{ type: "text", text: string }` → render as formatted text
- `{ type: "tool-invocation", toolInvocation: ToolInvocation }` → render as GenerativeCard

---

### ToolInvocation

Describes a single tool call within an assistant message. Provided by the AI SDK.

```
interface ToolInvocation {
  toolCallId: string
  toolName: GenerativeComponentType
  args: object                   // Typed per tool — always empty (data from resume.ts)
  state: "call" | "partial-call" | "result"
  result?: GenerativeToolResult  // Present when state === "result"
}
```

---

### GenerativeComponentType

Union of all tool names that map to a rendered React component.

```
type GenerativeComponentType =
  | "renderProjectCard"
  | "renderCareerTimeline"
  | "renderSkillTags"
  | "renderContactCard"
```

---

### GenerativeToolResult

Typed result payloads returned by each tool's `execute` function on the server.
Data is always sourced directly from `RESUME_DATA` (resume.ts).

```
// renderProjectCard result
interface ProjectCardResult {
  projects: Array<{
    name: string
    status: string
    description: string
    techStack: string[]
  }>
}

// renderCareerTimeline result
interface CareerTimelineResult {
  experience: Array<{
    company: string
    role: string
    period: string
    location: string
    description: string
    achievements: string[]
    techStack: string[]
  }>
}

// renderSkillTags result
interface SkillTagsResult {
  skills: {
    proficient: string[]
    intermediate: string[]
    knowledge: string[]
  }
}

// renderContactCard result
interface ContactCardResult {
  contact: {
    email: string
    linkedin: string
    location: string
    availability: string
  }
  // Note: phone is intentionally excluded (stored as env variable)
}
```

### SuggestionChip

A pre-filled question displayed in the Chat welcome state. Clicking one submits the
`prompt` field as a user message.

```
interface SuggestionChip {
  id: string
  label: string    // Display text on the chip button
  prompt: string   // The full question submitted when clicked
}
```

Default chips (defined as a static constant, not from resume.ts):

| id | label | prompt |
|---|---|---|
| `chip-stack` | `What's your tech stack?` | `What technologies are you most proficient with?` |
| `chip-projects` | `Show me your projects` | `Tell me about your projects` |
| `chip-experience` | `Work experience` | `Walk me through your career history` |
| `chip-contact` | `How to reach you` | `How can I contact Fernando?` |

---

### ChatSession

The in-memory conversation state for the current page visit. Managed entirely by
`useChat` — not serialised to any storage.

```
interface ChatSession {
  messages: ChatMessage[]   // Full ordered conversation history
  isLoading: boolean        // True while awaiting a streaming response
  error?: Error             // Set when the API request fails
}
```

Sessions are independent per mode: switching to Terminal and back does NOT reset
`ChatSession` (the `ChatInterface` component is kept mounted but hidden when in
Terminal mode).

## State Transitions

```
Portfolio
  ├── mode: "terminal"  →  (user clicks "Chat")  →  mode: "chat"
  └── mode: "chat"      →  (user clicks "Terminal") → mode: "terminal"

ChatSession.messages
  ├── [] (initial)
  ├── + SuggestionChip submitted → [{ role: "user", content: chip.prompt }]
  ├── + API response streaming → [..., { role: "assistant", parts: [...] }]
  └── (never reset on mode switch — component stays mounted)

ToolInvocation.state
  ├── "call" (tool invoked, no result yet)
  ├── "partial-call" (streaming args — rare for these parameterless tools)
  └── "result" (execute() returned data → render GenerativeCard)
```

---

## Visibility Rules

| Condition | Shown |
|---|---|
| `mode === "terminal"` | `TerminalView`; `ChatInterface` hidden (`display: none` equivalent, not unmounted) |
| `mode === "chat"` | `ChatInterface`; `TerminalView` hidden |
| `ChatSession.messages.length === 0` | `SuggestionChips` welcome block |
| `ChatSession.messages.length > 0` | Message list; `SuggestionChips` hidden |
| `toolInvocation.state === "result"` | Corresponding GenerativeCard component |
| `toolInvocation.state !== "result"` | Loading skeleton or nothing |
