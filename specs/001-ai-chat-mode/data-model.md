# Data Model: AI Chat Mode with Generative UI

**Feature**: 001-ai-chat-mode
**Date**: 2026-03-28 (updated 2026-03-30)

---

## Entities

### ChatMessage

A single message in the conversation. Follows the Vercel AI SDK `UIMessage` type so it
can be passed directly to `useChat` and serialised for the API route.

```
interface ChatMessage {
  id: string               // Unique identifier (generated by AI SDK)
  role: "user" | "assistant"
  parts: MessagePart[]     // Structured content parts (text, tool entries)
  createdAt?: Date
}
```

The `parts` array is the primary rendering target in `ChatMessage.tsx`:
- `{ type: "text", text: string }` → render as formatted text
- `{ type: "tool-*", ... }` → render as GenerativeCard when state is `"output-available"`

---

### ToolInvocation

Describes a single tool entry within an assistant message. Provided by the AI SDK on
`message.parts` as a part with `type` starting with `"tool-"`.

```
interface ToolPart {
  type: string              // e.g. "tool-renderProjectCard"
  toolCallId: string
  state: "input-available" | "input-streaming" | "output-available"
  output?: Record<string, unknown>  // Present when state === "output-available"
}
```

---

### GenerativeComponentType

Union of all tool names that map to a rendered React component.

```
type GenerativeComponentType =
  | "renderProjectCard"
  | "renderCareerTimeline"
  | "renderSkillTags"
  | "renderContactCard"
```

---

### GenerativeToolResult

Typed result payloads returned by each tool's `execute` function on the server.
Data is always sourced directly from `RESUME_DATA` (resume.ts).

```
// renderProjectCard result
interface ProjectCardResult {
  projects: Array<{
    name: string
    status: string
    description: string
    techStack: string[]
  }>
}

// renderCareerTimeline result
interface CareerTimelineResult {
  experience: Array<{
    company: string
    role: string
    period: string
    location: string
    description: string
    achievements: string[]
  }>
}

// renderSkillTags result
interface SkillTagsResult {
  skills: {
    proficient: string[]
    intermediate: string[]
    knowledge: string[]
  }
}

// renderContactCard result
interface ContactCardResult {
  contact: {
    email: string
    linkedin: string
    location: string
    availability: string
  }
  // Note: phone is intentionally excluded (stored as env variable)
}
```

### SuggestionChip

A pre-filled question displayed in the chat welcome state. Clicking one submits the
`prompt` field as a user message.

```
interface SuggestionChip {
  id: string
  label: string    // Display text on the chip button
  prompt: string   // The full question submitted when clicked
}
```

Default chips (defined as a static constant in `SuggestionChips.tsx`):

| id | label | prompt |
|---|---|---|
| `chip-stack` | `What's your tech stack?` | `What technologies are you most proficient with?` |
| `chip-projects` | `Show me your projects` | `Tell me about your projects` |
| `chip-experience` | `Work experience` | `Walk me through your career history` |
| `chip-contact` | `How to reach you` | `How can I contact Fernando?` |

---

### ChatSession

The in-memory conversation state for a single `/chat` command invocation. Managed
entirely by `useChat` in `ChatInterface` — not serialised to any storage.

```
interface ChatSession {
  messages: ChatMessage[]   // Full ordered conversation history
  isLoading: boolean        // True while awaiting a streaming response
  error?: Error             // Set when the API request fails
}
```

Each time the visitor runs `/chat`, a new `ChatInterface` mounts and a fresh `ChatSession`
begins. Sessions are not shared across multiple `/chat` entries in the terminal history.

---

## State Transitions

```
Terminal history (page.tsx)
  ├── user types "/chat" → history gains command + ChatInterface output item
  └── user types "/chat" again → second independent ChatInterface added to history

ChatSession.messages (per ChatInterface instance)
  ├── [] (initial — SuggestionChips visible)
  ├── + chip clicked / message submitted → [{ role: "user", parts: [{type:"text",...}] }]
  ├── + API response streaming → [..., { role: "assistant", parts: [...] }]
  └── (reset only if the page is refreshed)

ToolPart.state
  ├── "input-available" / "input-streaming" → show "thinking…" indicator
  └── "output-available" → render matching GenerativeCard component
```

---

## Visibility Rules

| Condition | Shown |
|---|---|
| `/chat` typed in terminal | `ChatInterface` appended to terminal history as command output |
| `ChatSession.messages.length === 0` | Welcome text + `SuggestionChips` |
| `ChatSession.messages.length > 0` | Message list; welcome block hidden |
| `toolPart.state === "output-available"` | Matching GenerativeCard component |
| `toolPart.state !== "output-available"` | `"thinking…"` pulse indicator |
